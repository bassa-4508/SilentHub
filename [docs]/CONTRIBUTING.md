# Contributing to SilentHub

Thanks for your interest in contributing.

## Rules

- Follow MVVM + Clean Architecture
- Keep files under 500 lines
- One responsibility per file
- UI and logic must be separated

## Flow

1. Fork repository
2. Create branch
3. Make changes
4. Submit PR

## Coding Style

- Kotlin standard conventions
- Use StateFlow for state
- Avoid direct file access from UI

## Notes

This project is primarily personal, but contributions are welcome.
